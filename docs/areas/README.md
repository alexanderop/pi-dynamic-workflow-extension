# Areas — Living Standards & Contracts

Docs maintained for as long as the code lives. No finish line: you keep these
current as the codebase changes, unlike [`../projects/`](../projects/) (which
have a goal and get archived when done).

- [`adr/`](./adr/) — architecture decision records. Durable design choices.
- [`spec-coverage.md`](./spec-coverage.md) — maps `spec.md` sections to the
  production modules and tests that own them. The "what's implemented" contract;
  for "what's next" see [`../projects/README.md`](../projects/README.md).
- [`references/`](./references/) — stable reference material:
  - `pi-extension-reference.md` — how this extension binds to Pi.
  - `testing-reference.md` — test conventions.
  - `error-handling.md` — error-type conventions.
  - `claude-code-workflow-tool.md` — the Claude Code workflow tool contract we mirror.
